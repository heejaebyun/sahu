## 마케팅 팀장
## 마케팅 팀장 의견

**3번 재방문** — 체크리스트는 본질적으로 일회성입니다. 재방문을 만들려면 "시간 기반 리마인더"가 핵심입니다. 사망일 입력 시 카카오 알림톡/이메일로 "상속세 신고 D-30", "재산분할 기한 D-7" 등 푸시를 보내면 자연스럽게 재방문이 발생합니다. 이건 제품 기능이지만 마케팅 채널(카톡)을 동시에 확보하는 구조입니다.

**6번 성장 채널** — SEO는 유지하되 단기 트래픽은 아래 3개에 집중:
1. **네이버 카페** — 장례/상속 관련 카페(예: 부동산스터디, 세무상담)에 "사망 후 행정절차 정리본" 형태로 자연 유입 유도
2. **카카오 오픈채팅/단톡방** — 장례 경험자 커뮤니티에 체크리스트 링크 공유 (유가족은 단톡방에서 정보 나눔)
3. **유튜브 쇼츠/릴스** — "가족이 돌아가시면 7일 안에 해야 할 것" 같은 15초 영상, 제작비 0원

**나머지 빠르게:**
- 4번(공유): 카톡 공유 버튼 하나면 충분, 이번 주 내 추가
- 5번(피드백): 체크리스트 하단에 "빠진 항목 있나요?" 구글폼 링크 1줄 추가
- 2번(다음 단계): 세무사 연결보다 먼저 "이 단계에서 많이 하는 실수" 콘텐츠로 체류시간 확보 → 이후 제휴 연결

**우선순위: 카톡 공유 버튼 → 리마인더 알림톡 → 카페/커뮤니티 시딩** 순으로 실행하겠습니다.

## CTO
## CTO 의견

**1. UX** — 현재 체크리스트를 확인해봐야 정확한 판단이 가능하지만, 핵심은 **모바일에서 한 손 조작 가능한 단일 컬럼 레이아웃 + 완료 항목 접기**입니다. 체크 시 localStorage 저장으로 새로고침해도 상태 유지 필수.

**4. 공유** — URL에 상태를 인코딩하는 방식(`?d=2026-03-01&done=1,3,5`)으로 구현하면 DB 없이 카카오톡/문자로 공유 가능. "카카오톡으로 공유" 버튼 하나면 충분. 구현 난이도 낮고 효과 큼.

**5. 피드백** — 체크리스트 하단에 "이 정보가 도움이 되었나요? 👍👎" 원클릭 반응 + 선택 시 한 줄 텍스트 입력 노출. Google Forms나 간단한 API route(`/api/feedback`)로 수집, 비용 0원.

**2~3, 6~7 요약:**
- **2번**: 체크리스트 완료 시 "세무사 무료 상담 연결" CTA → 이게 향후 매출 경로
- **3번**: 재방문은 공유 기능 + 체크 상태 저장이 해결. 추가로 D+30/90 리마인더(카카오 알림톡) 고려
- **6번**: SEO 콘텐츠가 1순위 맞음. 블로그 자동 발행 파이프라인 이미 scripts에 있으니 이걸 주 3회로 돌리는 게 가장 ROI 높음
- **7번**: 지금은 **공유 → 피드백 → 세무사 CTA** 순서로 1~2주 내 배포. 코드 복잡도 낮은 것부터.

구현 우선순위: **공유 버튼(1일) → 피드백 위젯(1일) → 체크 상태 저장(반나절) → 세무사 CTA(반나절)**. 이번 주 안에 다 가능합니다.

## 전략 이사
## SAHU 전략 의견

**핵심 판단: 지금은 "넓히기"가 아니라 "깊이 파기" 단계.**

1. **UX** — 모바일 퍼스트로 체크리스트 한 화면에 3개 이하만 노출. 스크롤 피로 제거가 급선무.
2. **다음 단계(핵심)** — 체크리스트 완료 시점에 "이 단계, 전문가에게 맡기기" CTA 삽입. 세무사/법무사 매칭이 아니라 **상담 신청 폼(카카오톡 채널)** 하나만 먼저 붙여라. 수요 검증이 매칭 플랫폼보다 선행.
3. **재방문** — 체크리스트를 "진행률 저장" 가능하게 만들어라. 로그인 없이 URL 토큰 방식. 이것이 곧 4번 공유 기능이 된다(링크 하나로 가족 간 진행 상황 공유).
4. **피드백** — 체크리스트 마지막에 "가장 어려웠던 절차는?" 1문항 설문. NPS보다 구체적 데이터 확보.
5. **SEO 외 성장** — 네이버 지식iN 차단됐으면 **카페**(상속/부동산 카페) 진입. "사망 후 행정 처리 순서" 정보글로 sahu.kr 유입 유도.
6. **차별화** — 경쟁사(장례 플랫폼)는 장례 "전"에 집중. SAHU는 장례 "후" 행정에 특화. 이 포지셔닝 절대 흐리지 말 것.
7. **성장 로드맵** — Phase 1 목표를 "월 1,000 UU + 상담 신청 50건"으로 잡고, 상담 전환이 입증되면 그때 세무사 제휴(수수료 모델)로 Phase 2 진입.

**한 줄 요약: 지금 만들 건 플랫폼이 아니라 "진행률 저장 + 상담 연결"이 되는 체크리스트 하나.**

## 비즈니스 디렉터
## SAHU 비즈니스 의견

**핵심: 지금은 수익화보다 "트래픽 → 리드 전환" 파이프라인 구축이 먼저.**

1. **세무사 연결 (2번)**: 체크리스트 완료 시 "상속세 신고, 혼자 하기 어렵다면?" CTA → 세무사/법무사 상담 신청 폼 하나만 넣으세요. 실제 연결은 수동으로 해도 됩니다. 이게 첫 매출 경로입니다. 제휴 수수료 or 건당 소개비 모델.

2. **프로덕트 성장 (7번)**: 체크리스트 자체는 "입구"일 뿐입니다. 성장 루프는 **체크리스트 → 상담 신청 → 세무사 매칭 → 후기 → SEO 콘텐츠 → 신규 유입** 순환입니다. 지금 빠진 건 "상담 신청" 한 칸뿐이니 이것부터 붙이세요.

3. **나머지 우선순위 정리**:
   - **(5) 피드백**: 체크리스트 하단에 카카오톡 오픈채팅 링크 하나면 충분. 도구 만들지 마세요.
   - **(4) 공유**: 카카오 공유 버튼 1개. 이건 반나절 개발.
   - **(3) 재방문**: 지금 단계에선 재방문보다 신규 유입 볼륨이 중요. 재방문은 Phase 2 문제.
   - **(6) 채널**: 네이버 블로그 주 3회 발행이 SEO 다음으로 ROI 높음. 지식인 차단은 새 계정으로 우회.
   - **(1) UX/모바일**: 실사용자 5명한테 화면 녹화 받아보고 판단. 추측으로 고치지 마세요.

**한 줄 요약: 이번 주 안에 "세무사 상담 신청 폼" 하나 붙이고, 세무사 2-3곳 제휴 연락하세요. 그게 Phase 1의 전부입니다.**

## 4. 핵심 메시지
**"이번 주 안에 공유 버튼과 상담 신청 폼, 딱 2개만 붙여라. 나머지는 전부 다음 주 이후다."** 기능을 넓히지 말고, 체크리스트 → 상담 신청 → 세무사 연결, 이 한 줄기 파이프라인을 먼저 관통시켜라.
